# Second Brand OS Onboarding Map

## 1. Baca Dulu

Urutan onboarding:
1. `AGENTS.md`
2. `core/SOUL.md`
3. `core/USER.md`
4. `hermes.md`
5. `docs/SECOND_BRAND_OS_BLUEPRINT.md`
6. `docs/INBOX_ROUTING.md`
7. `docs/TOPIC_WORKSPACE_INDEX.md`
8. `docs/SECOND_BRAND_OS_SOP_PER_LANE.md`
9. `docs/SECOND_BRAND_OS_VISI_MISI.md`

## 2. Peta Cepat Folder

| Folder | Fungsi | Gunakan saat |
|---|---|---|
| `core/` | identitas dan aturan dasar | boot, tone, user model |
| `docs/` | governance dan blueprint | butuh source of truth |
| `memory/` | continuity harian | catat kejadian penting |
| `knowledge-base/` | source dan canon | simpan pengetahuan reusable |
| `brand-os/` | strategi brand | positioning, offer, voice |
| `personal-brand/` | output publik | konten dan asset |
| `clients/` | delivery klien | brief dan pekerjaan klien |
| `projects/` | proyek internal | eksperimen dan tool lab |
| `research/` | riset aktif | benchmark dan analysis |
| `ops/` | runtime dan debug | automation dan troubleshooting |
| `automation/` | jobs berulang | cron, reminder, scheduler |
| `wellbeing/` | sustainabilitas manusia | energi, ritme, habit |
| `archives/` | legacy dan snapshot | referensi historis |

## 3. Peta Cepat Topic Telegram

| Topic | Fungsi singkat |
|---|---|
| `Command Center` | keputusan besar dan brainstorming lintas lane |
| `INBOX` | capture awal dan routing |
| `tasks` | kerja aktif dan eksekusi |
| `content` | draft dan output publik |
| `personal-crm` | relasi, follow-up, deal |
| `knowledge-base` | source, ringkasan, knowledge |
| `ops` | runtime, infra, debug |
| `updates` | summary, alert, progress |
| `Hormozi` | offer, pricing, funnel, sales copy |

## 4. Alur Kerja Harian

1. Terima input.
2. Tentukan lane.
3. Route ke topic atau folder.
4. Ringkas jika perlu.
5. Simpan ke memory atau knowledge-base jika reusable.
6. Eksekusi di lane tujuan.
7. Archive jika selesai.

## 5. Rules yang Harus Diingat

- Inbox hanya capture.
- Knowledge mentah bukan canon.
- Output publik tidak disimpan sebagai source.
- Runtime harus auditable.
- Routing harus eksplisit.
- Hormozi dipakai hanya untuk offer/pricing/funnel/sales-copy work.

## 6. Jika Bingung

Pakai urutan ini:
- mentah → `INBOX`
- sumber → `knowledge-base`
- kerja aktif → `tasks`
- output publik → `content`
- orang → `personal-crm`
- sistem → `ops`
- strategi besar → `Command Center`
- offer/pricing/funnel → `Hormozi`
