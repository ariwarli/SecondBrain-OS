# Hormozi Workspace Guide

Panduan ini menjelaskan cara memakai topic Telegram `Hormozi` sebagai lane khusus untuk framework bisnis, offer, pricing, funnel, sales copy, dan knowledge synthesis.

## Tujuan Topic Hormozi

Topic `Hormozi` dipakai sebagai workbench untuk:
- menyerap source mentah
- mengekstrak konsep dan framework
- menyusun offer, pricing, dan positioning
- menulis hook, angle, dan sales copy
- memisahkan knowledge bisnis dari output publik dan eksekusi tugas

## Kapan Dipakai

Gunakan topic `Hormozi` kalau pesan terkait:
- offer
- pricing
- funnel
- value equation
- acquisition
- lead magnet
- sales copy
- conversion strategy
- source-to-concept synthesis

Jangan pakai topic ini kalau pesan utamanya:
- kerja teknis runtime
- follow-up personal
- task operasional umum
- draft konten biasa tanpa konteks Hormozi
- intake mentah yang belum jelas relevansinya

## Peran dalam Workspace

### 1. `Hormozi`

Lane kerja khusus. Di sini kamu minta:
- klasifikasi source
- ringkasan framework
- mapping konsep
- ide offer
- ide pricing
- angle konten berbasis framework

### 2. `knowledge-base`

Lane library dan intake. Dipakai untuk:
- simpan source mentah
- simpan ringkasan
- promosi ke wiki canon
- catatan yang belum jadi output

### 3. `content`

Lane output. Dipakai untuk:
- post
- carousel
- thread
- script
- hook
- caption

### 4. `tasks`

Lane eksekusi. Dipakai untuk:
- audit
- riset lanjutan
- revisi
- compile
- implementasi

## Alur Kerja Disarankan

1. Kirim bahan mentah ke `Hormozi`
2. Minta klasifikasi: `source`, `concept`, atau `entity`
3. Minta turunkan jadi framework, offer, atau angle
4. Simpan hasil matang ke `knowledge-base` atau wiki canon
5. Jika mau dipublikasikan, pindahkan ke `content`
6. Jika perlu dikerjakan, pindahkan ke `tasks`

## Format Prompt Siap Pakai

### Untuk klasifikasi

- `Klasifikasikan ini jadi source, concept, atau entity.`
- `Tolong ringkas dan tentukan bagian yang bisa jadi framework.`
- `Ambil inti Hormozi dari materi ini.`

### Untuk synthesis

- `Ubah source ini jadi konsep reusable.`
- `Turunkan jadi framework bisnis yang jelas.`
- `Bikin versi yang bisa dipakai ulang di workspace.`

### Untuk offer dan pricing

- `Analisis ini pakai lens Hormozi untuk offer.`
- `Bikin value equation dari produk ini.`
- `Tolong bantu susun pricing logic yang lebih kuat.`

### Untuk content

- `Bikin 5 hook dari framework ini.`
- `Ubah ini jadi LinkedIn post.`
- `Buat carousel 7 slide dari konsep ini.`
- `Tulis script pendek dari materi ini.`

## Aturan Routing

- `Hormozi` = workbench
- `knowledge-base` = library
- `content` = output
- `tasks` = action
- `Inbox` = capture saja, jangan kerja di sana

Kalau topik Hormozi sangat sering dipakai, pertahankan sebagai lane khusus. Kalau pemakaian rendah, cukup jadikan domain knowledge tanpa memperbanyak topik.

## Promosi ke Canon

Naikkan hasil dari `Hormozi` ke wiki hanya bila sudah punya nilai reusable:
- `Research` untuk source summary atau synthesis
- `Frameworks` untuk model pikir atau rubric
- `SOPs` untuk langkah kerja konsisten
- `Decisions` untuk keputusan final
- `Incidents` untuk failure, root cause, atau lesson

## Contoh Alur Nyata

### Kasus 1: kirim artikel
- Kirim link ke `Hormozi`
- Minta ringkasan
- Minta konsep inti
- Simpan hasil ke `knowledge-base`

### Kasus 2: mau bikin offer
- Kirim deskripsi produk ke `Hormozi`
- Minta value equation
- Minta positioning
- Minta draft offer
- Lanjutkan ke `content` atau `tasks`

### Kasus 3: mau bikin konten
- Kirim framework ke `Hormozi`
- Minta 5 hook
- Minta post atau carousel
- Pindahkan output final ke `content`

## Ringkasan Cepat

Kalau ragu:
- sumber mentah masuk `Hormozi`
- hasil belajar masuk `knowledge-base`
- hasil publik masuk `content`
- pekerjaan aktif masuk `tasks`
