# Claude Workflow Checklist — Threads Draft

Angle: **"Stop using Claude like a chatbot. Build a real work system."**
Tone: Langsung, praktis, gw/lo/bro playbook
Target: AI consultant, founder, content creator yang stuck di "chat mode"

---

**[POST 1] HOOK**

Gw baru ketemu framework Claude yang bikin nyesel kenapa gak nemu ini dari dulu.

Most people pake Claude kayak chatbot ngobrol random.

Padahal dia bisa jadi WORK SYSTEM yang auto-pilot your whole day.

Ini 9 step buat beralih dari "chat" ke "system". 🧵

---

**[POST 2] FASE 1: FOUNDATION**

Dulu gw juga langsung nanya-nanya tanpa prepare. Hasilnya? Random & gak konsisten.

Yang bener → bayar (serius, worth it), download desktop app, turn on "stronger thinking" buat task berat, pelajari platform dulu.

You can't build a house on sand.

---

**[POST 3] FASE 2: WORKSPACE SETUP**

One folder. Semua disitu.

Main workspace → subfolder: context, outputs, templates.

Tulis "about me" file. Biar Claude ngerti WHO you are.

Claude itu hafal limited. Kalau folder rapi, dia kerja 10x lebih fokus.

---

**[POST 4] FASE 3: BETTER PROMPTING**

Ini yang bikin beda amatir vs pro:

→ Explain what you want (jelasin, jangan asal)
→ Define success (metriknya apa?)
→ Ask Claude to clarify DULU sebelum mulai

Game changer: "Please ask me clarifying questions before you start."

---

**[POST 5] FASE 4-5: TOOLS + REPEATABLE SYSTEMS**

Claude connect ke Gmail, Drive, Notion, GitHub.

Rules: Connect ONLY yang useful. Turn off yang gak perlu. Less is more.

Terus build:
→ Reusable commands
→ Saved workflows
→ "Do NOT do" rules
→ 1 project = 1 recurring task

Integrasi bukan buat pamer. Buat execute.

---

**[POST 6] FASE 6-7: PROJECTS + DESIGN**

Upload best examples jadi reference. Save templates. Build repeatable delivery.

Terus pake buat:
→ Landing pages
→ Content systems
→ Brand structure

Faster execution. Lebih sedikit revisi.

---

**[POST 7] FASE 8-9: RESEARCH + TOKEN**

Smarter research:
→ Web search, multiple searches
→ Find info gaps sebelum publish
→ Verify facts

Token management:
→ Plan first, build second
→ Keep files lightweight
→ Restart chat kalau overload

Restart chat = feature, bukan failure.

---

**[POST 8] THE QUOTE**

"Stop using Claude like a chatbot. Use it like a real work system."

Structure kills randomness.

Most people cari model AI lebih canggih.
Few people build system yang make AI work 10x harder.

Sistem > AI.

---

**[POST 9] CTA**

Gw udah apply sebagian ini ke workflow sendiri.

Hasilnya? 3-4x lebih cepet deliver client, lebih sedikit revisi, gak capek ngulang-ngulang brief yang sama.

File "about me" + template prompts gw save di Obsidian.

Kalau lo udah coba beberapa dari 9 ini, reply angkanya. Mau tau mana yang paling ngefek.
