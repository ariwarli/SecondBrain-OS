# Daily Operating Rhythm

Dokumen ini adalah ritme kerja praktis untuk memakai REED dan Second Brain setiap hari.

Gunakan ini kalau Anda tidak ingin memikirkan sistemnya terlalu banyak, tetapi tetap ingin hasilnya rapi dan compounding.

## Prinsip Dasar

- Telegram untuk menangkap
- REED untuk merapikan
- Obsidian untuk menyimpan
- `*-active.md` untuk konteks aktif
- checkpoint biasa untuk histori

## Pagi

Tujuan pagi:

- buka konteks aktif
- tahu prioritas
- tahu blocker
- hindari mulai kerja tanpa orientasi

### Langkah pagi

1. buka topic yang relevan
2. kirim template start kerja
3. baca ringkasan keputusan, blocker, dan next action
4. mulai dari item paling penting

### Template pagi

```text
Gw mulai kerja sekarang. Buka konteks aktif untuk lane ini, ringkas status sekarang, lalu kasih:
1. keputusan aktif
2. blocker
3. next action paling penting
4. apa yang berubah terakhir kalau ada
```

## Siang

Tujuan siang:

- menangkap hal baru secepat mungkin
- tidak menyimpan semuanya di kepala
- memperbarui konteks kalau ada perubahan penting

### Kapan kirim update

- habis meeting
- dapat info baru dari klien
- ada keputusan baru
- ada blocker baru
- ada ide yang layak disimpan

### Template siang

```text
Update konteks:
[tulis update di sini]

Kalau ini mengubah keputusan, blocker, atau next action aktif, update memory aktifnya juga.
Kalau tidak, cukup simpan sebagai histori yang relevan.
```

## Sore

Tujuan sore:

- merapikan lane
- membuang yang sudah selesai
- memastikan hal penting tidak hilang

### Langkah sore

1. cek apakah lane masih bersih
2. minta REED rapikan konteks aktif
3. simpan sesi jika memang penting

### Template sore

```text
Rapikan lane ini.

Tolong:
- buang item yang sudah selesai atau usang dari konteks aktif
- pertahankan hanya keputusan, blocker, dan next action yang masih relevan
- bilang kalau ada gap knowledge yang perlu diisi
```

### Template save sesi penting

```text
Save upload session ini ke knowledge-base.

Ringkas hanya:
- keputusan
- fakta baru
- blocker
- next action

Kalau ini mengubah konteks aktif, update file aktifnya juga. Kalau tidak, cukup simpan sebagai histori sesi.
```

## Mingguan

Tujuan mingguan:

- melihat pola
- melihat apa yang mandek
- melihat area knowledge yang kosong
- menjaga active context tetap relevan

### Langkah mingguan

1. cek lane yang paling sering dipakai minggu ini
2. minta REED jalankan review mingguan
3. cek `knowledge-base/index.md`
4. cek `knowledge-base/wiki/log.md`
5. lihat apakah ada keputusan usang atau next action yang tertinggal

### Template mingguan

```text
Jalankan review mingguan untuk lane ini.

Tolong cek:
- apakah konteks aktif masih relevan
- apakah ada keputusan usang
- apakah ada next action yang mandek
- apakah ada hal penting yang belum masuk knowledge
```
